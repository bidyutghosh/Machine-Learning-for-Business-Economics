📌 Project Description

The Car Price Prediction project explores the fundamental factors that influence automobile valuations in the competitive automotive market. Utilizing a popular Kaggle Dataset containing extensive technical specifications, this analysis serves as a core case study for Supervised Machine Learning Regression.

🎯 Key Objectives

  • Price Estimation: Predict the market price of a vehicle based on its physical properties and mechanical capabilities.
  • Feature Evaluation: Analyze how categorical traits (e.g., body style, fuel type) and continuous stats (e.g., horsepower, engine size) affect price.
  • Strategic Value: Emulate a real-world business case where manufacturers optimize pricing strategy to maximize profitability.

📊 Variable Dictionary (1/2)

📋 Variable Name 🛠️ Data Type 📝 Feature Description
car_ID Integer Unique identifier for each car entry.
symboling Integer Insurance risk rating (varies from -3 safe to +3 risky).
CarName String Brand and specific model name of the vehicle.
fueltype String Type of fuel consumed (Gas or Diesel).
aspiration String Internal air intake system (Standard or Turbocharged).
doornumber String Number of doors on the vehicle (Two or Four).
carbody String Vehicle structural body style (Sedan, Hatchback, etc.).
drivewheel String Configuration of driven wheels (FWD, RWD, 4WD).
enginelocation String Physical placement of the motor (Front or Rear).
wheelbase Float Distance between front and rear wheel hubs.
carlength Float Total longitudinal length of the car.
carwidth Float Total lateral width of the car.
carheight Float Total vertical height of the car.

📊 Variable Dictionary (2/2)

📋 Variable Name 🛠️ Data Type 📝 Feature Description
curbweight Integer Total vehicle mass without passengers or payload.
enginetype String Structural engine blueprint (OHC, DOHC, OHCV, etc.).
cylindernumber String Textual count of engine cylinders (Four, Six, etc.).
enginesize Integer Engine displacement or volume capacity.
fuelsystem String Methodology of fuel delivery (MPFI, 2BBL, IDI, etc.).
boreratio Float Ratio of the cylinder bore diameter to piston stroke.
stroke Float Longitudinal distance travelled by the piston inside the cylinder.
compressionratio Float Volume ratio of combustion chamber (maximum vs minimum).
horsepower Integer Mechanical power output capacity of the engine.
peakrpm Integer Maximum engine revolutions per minute achieved.
citympg Integer Fuel efficiency rating under urban driving conditions.
highwaympg Integer Fuel efficiency rating under highway driving conditions.
price Float Target Variable: Total market valuation / selling price.

Summary of Exploratory Data Analysis

Here is a slide-ready, pointwise summary of the data exploration phase, framed specifically through the lens of preparing an ML regression model.

🎯 Target Variable Optimization

  • Skewness: The raw target variable price is heavily right-skewed.
  • Fix: Apply a log-transformation (\(\log(\text{price})\)).
  • Impact: Normalizes residuals and stabilizes variance (homoscedasticity).

📐 Numerical Feature Insights

  • Strong Predictors: enginesize, curbweight, and horsepower scale linearly with price.
  • Inverse Effects: Higher fuel efficiency (citympg, highwaympg) correlates with cheaper vehicles.
  • Weak Signals: Drop or transform carheight and stroke due to low linear correlation.

⚠️ Multicollinearity Management

  • Redundancy: Severe collinearity exists between citympg and highwaympg (\(0.97\)).
  • Dimensional Tie-ins: Physical dimensions (carlength, carwidth) heavily mirror curbweight.
  • Action: Drop one feature from highly correlated pairs to prevent erratic model coefficients.

🏷️ Categorical Feature Engineering

  • High Variance: enginelocation, drivewheel, and cylindernumber act as strong price segmentors.
  • Low Variance: Drop doornumber because it lacks predictive power.
  • Preprocessing: Use One-Hot Encoding for remaining categorical variables.

from sklearn.compose import ColumnTransformer
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler, OneHotEncoder

X = df.drop(columns=['price', 'doornumber'])  # Dropping low-impact doornumber
y = np.log(df['price'])                        # Log-transforming target

# 2. Identify Column Types Programmatically
num_cols = X.select_dtypes(include=['int64', 'float64']).columns.tolist()
cat_cols = X.select_dtypes(include=['object']).columns.tolist()
# Remove Car Nmaes from Categorical Features
cat_cols.remove('CarName')
# 3. Create Sub-Pipelines for Each Feature Type
num_transformer = Pipeline(steps=[
    ('scaler', StandardScaler())
])

cat_transformer = Pipeline(steps=[
    ('onehot', OneHotEncoder(drop='first', handle_unknown='ignore', sparse_output=False))
])

# 4. Combine Transformers using ColumnTransformer
preprocessor = ColumnTransformer(
    transformers=[
        ('num', num_transformer, num_cols),
        ('cat', cat_transformer, cat_cols)
    ])

# 5. Create Full End-to-End Modeling Pipeline
lr_pipeline = Pipeline(steps=[
    ('preprocessor', preprocessor),
    ('regressor', LinearRegression())

Model Evaluation

Code 
import numpy as np
from sklearn.metrics import r2_score, mean_squared_error, mean_absolute_error
def evaluate_model(model, X_train, y_train, X_test, y_test):
    # 1. Get predictions (Regular Minute Scale)
    y_train_pred = model.predict(X_train)
    y_test_pred = model.predict(X_test)
    
    # --- TRAINING DATA METRICS ---
    r2_train = r2_score(y_train, y_train_pred)
    mse_train = mean_squared_error(y_train, y_train_pred)
    rmse_train = np.sqrt(mse_train)
    mae_train = mean_absolute_error(y_train, y_train_pred)
    # Correct MAPE calculation
    mape_train = np.mean(np.abs((y_train - y_train_pred) / y_train)) * 100
    
    # --- TESTING DATA METRICS ---
    r2_test = r2_score(y_test, y_test_pred)
    mse_test = mean_squared_error(y_test, y_test_pred)
    rmse_test = np.sqrt(mse_test)
    mae_test = mean_absolute_error(y_test, y_test_pred)
    # Correct MAPE calculation
    mape_test = np.mean(np.abs((y_test - y_test_pred) / y_test)) * 100
    
    print(f"Train Data: R-squared = {r2_train:.4f}, MSE = {mse_train:.4f}, RMSE = {rmse_train:.4f}, MAE = {mae_train:.4f}, MAPE = {mape_train:.2f}%")
    print(f"Test Data:  R-squared = {r2_test:.4f}, MSE = {mse_test:.4f}, RMSE = {rmse_test:.4f}, MAE = {mae_test:.4f}, MAPE = {mape_test:.2f}%")

  • Minimal Error: A test Mean Absolute Percentage Error (MAPE) of 1.47% indicates exceptionally tight price estimations.
  • Low Residual Spread: The overall test error remains consistently small (\(\text{RMSE} = 0.1657\)).

⚖️ Excellent Generalization (No Overfitting)

  • Tight Train-Test Alignment: Training accuracy (\(R^2 = 0.9439\)) closely mirrors testing accuracy (\(R^2 = 0.8988\)).
  • Robust Stability: The model successfully avoids memorizing noise, ensuring stable and reliable performance on completely new vehicle data.

⚠️ Pipeline Safe-Guard Warning

  • Encountered Reality: The OneHotEncoder flagged an unseen subcategory in the test set (Column index 6).
  • Silent Fix: The handle_unknown=‘ignore’ safety parameter encoded this new category as all zeros, preventing a fatal runtime crash without destabilizing overall accuracy.

Tree Based Models

🟢 Why Tree-Based Models Excel

  • Zero Scaling Needed: They completely bypass the need for StandardScaler transformations.
  • No Target Transformations: They comfortably handle raw, right-skewed pricing without log-transformations.
  • Non-Linear Relationships: They natively capture complex feature interactions automatically.
  • Outlier Robustness: Extreme luxury car outliers will not distort the decision splits.

🔴 Why They Risk Performance Here

  • Severe Overfitting Danger: Our sample size is incredibly small (\(N=205\)).
  • Data Scarcity: Complex architectures like XGBoost will instantly memorize this tiny dataset.
  • No Extrapolation: Trees cannot predict a price higher than the maximum value in training.
  • Black Box Loss: We lose the elegant, easily interpretable coefficients (\(e^\beta\)) of linear regression.

🎓 Recommendation

We use a Random Forest Regressor exclusively as a secondary baseline comparison model. We apply strict regularization parameters (max_depth=4, min_samples_leaf=5) to prevent overfitting. This provides an excellent teaching moment to compare linear models against ensemble architectures.

Hyperparameter Tuning of Random Forests

Code 
# %%
from sklearn.model_selection import GridSearchCV

# 1. Base Pipeline Setup (Reusing your original ColumnTransformer preprocessor)
rf_pipeline = Pipeline(steps=[
    ('preprocessor', preprocessor),
    ('regressor', RandomForestRegressor(random_state=42))
])

# 2. Define Hyperparameter Grid (Using 'regressor__' prefix to target the model step)
param_grid = {
    'regressor__n_estimators': [100, 200],
    'regressor__max_depth': [4, 6, 8],
    'regressor__min_samples_leaf': [2, 4, 6],
    'regressor__max_features': ['sqrt', 0.8]  # Helps reduce overfitting by restricting column access per split
}

# 3. Setup GridSearchCV with 5-Fold Cross Validation
grid_search = GridSearchCV(
    estimator=rf_pipeline,
    param_grid=param_grid,
    cv=5,
    scoring='neg_mean_absolute_percentage_error',  # Optimizes directly for lower MAPE
    n_jobs=1
)

# 4. Fit on Training Splits
grid_search.fit(X_train_t, y_train_t)

# 5. Extract Best Parameters and Best Score
print("Best Parameters:", grid_search.best_params_)
print("Best CV Score (MAPE):", abs(grid_search.best_score_))
